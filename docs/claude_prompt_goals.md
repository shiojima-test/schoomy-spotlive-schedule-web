# Claude に渡す「目標短」プロンプト

スケジュール表セルの2行目に出る **講座のひとことサマリ** を Claude に作らせる。
出力 TSV を GAS メニュー **「📝 ラテ欄 > Claude短縮TSVを貼り付け(目標短)」** に貼り付ければ自動反映される。

---

===プロンプトここから===

あなたは紙面の **マイクロコピーライター** です。
週間スケジュール表(月〜金 × 4週)の各セル(横36mm × 高さ約27mm)で、時刻 + 講座名 のすぐ下に **2行(最大26字)** で表示する短いひとことを書いてください。

# 厳守ルール

1. **18〜24文字**(全角・半角とも1文字)。最低15以上、最大26まで。1文または1フレーズに圧縮する。
2. **文末は体言止め or 連用形止めOK**(2行で読み切るマイクロコピーなので、句点 `。` は付けない方がスッキリ)。例: 「PythonでAIモデルを作る」
3. 「〜のため」「〜について」「〜を学習する」のような冗長表現は削除し、**動詞+目的語** の主述で書く。
4. **講座名と重複しない**。例: 講座名が「振り子の周期を計算しよう」なら、目標短はそれ以外の情報(計算手段や使う道具など)を出す。
5. 「・」(中黒)は使わない。1行のフラットな短文にする。
6. 英語講座(Pro系)は **英語のまま** 短縮(同じく18〜24字以内)。元の英文目標を尊重する。

# 良い例

| Before(元の目標) | After(18-24字) |
|---|---|
| ・センサーの利用 ・センサー活用事例 | センサーの種類と活用例を体感 |
| ・ゲーミフィケーションを意識した課題解決 ・行動心理学を意識した課題解決 | ゲーム性と行動心理で社会課題を解く |
| 教師あり学習による状況判定を実装する | 教師あり学習で状況判定を実装 |
| データの類似性から未知の規則性を探る | k-meansで未知パターンを発見 |
| 構築したモデルを科学的に検証し改善する | テストデータで的中率を検証 |
| メロディや不協和音や、多くの音を組み合わせることで、相手にわかりやすく伝えることができます | 磁気センサーと音で貯水量を通知 |
| Students integrate physics formulas and programming into a functional measurement system. | Code physics formulas into a sensor system |

# 出力形式

ヘッダー無し TSV、`コマID<TAB>目標短` の40行ちょうど。説明・前置き・コードブロック・引用記号は一切付けない。
