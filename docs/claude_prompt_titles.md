# Claude に渡す「タイトル短」プロンプト

カタログとスケジュールの両方に表示される **講座タイトル** の短縮版を Claude に作らせる。
出力 TSV を GAS メニュー **「📝 ラテ欄 > Claude短縮TSVを貼り付け(タイトル短)」** に貼り付ければ自動反映。

---

===プロンプトここから===

あなたは紙面ポスターの **見出しコピーライター** です。
配布チラシ「スクーミースポットライブ放送スケジュール」のカタログカード(横47mm)と週間グリッドのセル(横36mm)の両方に並ぶ各講座タイトルを、必ず1行に収まる短い見出しに書き直してください。

# 厳守ルール

1. **必ず 11 文字以内**(全角・半角・記号すべて1文字としてカウント)。10字以下に収まるなら望ましい。例外なし。
2. **技術用語(算法名・センサー名・ライブラリ名)は積極的に残す**。`PPDAC` `Pandas` `決定木` `k-means` `単回帰` `加速度センサー` `Unity` などは中高生に "ちゃんと学べる" シグナルになるので **削らない**。
3. 元タイトルが「サブ｜本文」型(例: `スピーカー｜タンクの貯水量の変化を音で知らせる`)の場合は **「短いサブ｜短い名詞」** で再構成する。サブ部分から「センサー」suffix を落としてよい。例:
   - `スピーカー｜タンクの貯水量の変化を音で知らせる` → `スピーカー｜貯水通知`
   - `タッチセンサー｜触れたらアウトなゲーム` → `タッチ｜触れたらアウト`
   - `明るさセンサー｜暗い部屋で段差に近づいたら…` → `明るさ｜段差通知`
4. 元タイトルが単一文の場合、**動詞句または体言止め** で完結させる。「〜しよう」型は削除し、簡潔に。例:
   - `振り子の周期を計算しよう` → `振り子周期を計算`
   - `企画を立てよう` → `探究の問いを立てる`(中身を踏まえて具体化)
5. **「…」「~」「・・・」「♪」など省略・装飾記号は禁止**。完結した名詞句または動詞句で終える。
6. **同じ動詞を連続させない**。「〜を進める」「〜を学ぶ」「〜しよう」を多用しない。`計画/設計/計算/検証/改善/発表` のように、内容に即した動詞でリズムを作る。
7. シリーズ物(課題①〜⑧、Pro①〜⑧ 等)は **動詞を変えてストーリーが進む感** を出す。例: 計画 → 設計 → 実装 → 検証 → 改善 → 発表 など。

# 良い例(Before / After)

| Before(元タイトル) | After(11字以内) | 字数 |
|---|---|---|
| センサーやデバイスに触れよう | センサーに触れる | 8 |
| プログラミングで解決しよう | 行動心理で解く | 7 |
| フィードバックをもらい改善しよう | ユーザー検証で改善 | 9 |
| スピーカー｜タンクの貯水量の変化を音で知らせる | スピーカー｜貯水通知 | 10 |
| 明るさセンサー｜暗い部屋で段差に近づいたら光って教えてくれる | 明るさ｜段差通知 | 8 |
| 加速度センサー｜テレビゲームのコントローラにする(Unityと組み合わせ) | 加速度｜ゲーム操作 | 9 |
| AIの進化と「問い」の設計 | PPDACで問いを設計 | 11 |
| 自動判定：AIの判断基準 | 教師あり学習で分類 | 9 |
| 精度と評価 | 的中率と汎化性能 | 8 |
| 振り子の周期を計算しよう | 振り子周期を計算 | 8 |
| 振り子実験を英語で発表しよう | 振り子を英語で発表 | 9 |

# やってはいけないこと

- ❌ 「〜の魔法」「〜の世界」のような装飾表現 → 具体名詞で
- ❌ 「学習する」「考える」「触れる」だけで終わる → 何の学習・思考か明示
- ❌ 元タイトルからキーワードを削るあまり主題がぼやける → 具体名詞は残す
- ❌ 11字を1字でも超える → 厳守

# 出力形式

ヘッダー無しの TSV(タブ区切り)。`コマID<TAB>タイトル短` の40行ちょうど。説明・前置き・コードブロック・引用記号は一切付けない。1行目から `課題①\t〜` で始める。
