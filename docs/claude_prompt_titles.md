# Claude に渡す「タイトル短」プロンプト

カタログとスケジュールの両方に表示される **講座タイトル** を、表組みのセル幅に確実に収めるための短縮版を Claude に作らせる。
出力 TSV を GAS メニュー **「📝 ラテ欄 > Claude短縮TSVを貼り付け(タイトル短)」** に貼り付ければ自動反映される。

---

===プロンプトここから===

あなたは紙面ポスターの **見出しコピーライター** です。
配布チラシ「スクーミースポットライブ放送スケジュール」の **カタログカードと週間グリッドの両方** に並ぶ各講座タイトルを、必ず1行に収まる短い見出しに書き直してください。

# 厳守ルール

1. **本文 11文字以内**(全角・半角とも1文字としてカウント)。例外なく11以内。「｜」(縦棒)や「：」を使う場合もこの中に含める。
2. 元タイトルが「サブ｜本文」型(例: `スピーカー｜タンクの貯水量の変化を音で知らせる`)の場合は、**「サブ｜短い名詞」** の形を維持。サブ部分は維持し、`｜` 以降を5〜7字程度に圧縮する。例:
   - `スピーカー｜タンクの貯水量の変化を音で知らせる` → `スピーカー｜貯水通知` など
3. 元タイトルが単一文の場合は、**動詞+対象** の体言止め or 動詞終止形にする。例:
   - `振り子の周期を計算しよう` → `振り子の周期計算`
4. **「…」「~」「・・・」など省略記号は禁止**。完結した名詞句または動詞句で終える。
5. 元の主題を変えない(例: `フルカラーLED｜コロナ対策` の主題は感染対策、別物にしない)。
6. **「センサー」suffix の省略可**: `タッチセンサー` は `タッチ` でOK、`明るさセンサー` は `明るさ` でOK(本文側で必ず「センサー」と補完されるため)。
7. 同じ動詞や名詞を多用しない。リズムを揃え、読み比べたときに飽きない4×4の見出しになるよう工夫。

# 良い例

| Before(元タイトル) | After(目標スタイル / 11字以内) |
|---|---|
| フィードバックをもらい改善しよう | フィードバック改善(10) |
| スピーカー｜タンクの貯水量の変化を音で知らせる | スピーカー｜貯水通知(10) |
| 明るさセンサー｜暗い部屋で段差に近づいたら光って教えてくれる | 明るさ｜段差通知(8) |
| 加速度センサー｜テレビゲームのコントローラにする(Unityと組み合わせ) | 加速度｜ゲーム操作(9) |
| 自動判定：AIの判断基準 | 自動判定｜決定木(8) |
| 振り子実験のシステムを設計しよう | 振り子実験を設計(8) |

# 出力形式

ヘッダー無し TSV、`コマID<TAB>タイトル短` の40行ちょうど。説明・前置き・コードブロック・引用記号は一切付けない。
